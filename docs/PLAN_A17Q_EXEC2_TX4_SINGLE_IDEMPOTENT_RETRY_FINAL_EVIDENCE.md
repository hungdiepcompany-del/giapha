## Final verifier correction

The previous final verifier produced four false negatives:

```text
PEOPLE_ROWS_UNCHANGED_PASS=FALSE
LAYOUT_ROWS_UNCHANGED_PASS=FALSE
EXCLUDED_HASH_UNCHANGED_PASS=FALSE
DELETED_HASH_UNCHANGED_PASS=FALSE
```

Root cause:

- the verifier compared audit `after_hash` values with same-named keys inside
  persisted `success_result`;
- persisted `success_result` does not contain `people_after_hash`,
  `layout_after_hash`, `excluded_after_hash`, or `deleted_after_hash`;
- those hashes are stored in the pre/post audit revisions;
- the executor already enforces before commit that the after hashes match the
  before hashes for people, layout, excluded scope, and deleted-family scope;
- any mismatch would raise `A17Q_POST_STATE_VALIDATION_FAILED` and roll back.

Correction:

```text
VERIFIER_FALSE_NEGATIVE_PROVEN=YES
VERIFIER_PATCHED=YES
FOUR_AUDIT_HASH_PASSES_COMPARE_NONEMPTY_BEFORE_AFTER_AUDIT_HASHES=YES
FOUR_AUDIT_HASH_PASSES_DO_NOT_REFERENCE_NONEXISTENT_SUCCESS_RESULT_HASH_KEYS=YES
STORED_SUCCESS_RESULT_SHA_BATCH_DECISION_HASH_CHECKS_PRESERVED=YES
EXECUTOR_PRE_COMMIT_HASH_GUARDS_PRESERVED=YES
```
