The locked branch must reject execution if any condition is not satisfied:

- validation errors > 0;
- dry-run blockers > 0;
- duplicate unresolved > 0;
- duplicate needs_review > 0;
- manifest hash mismatch;
- `confirmSessionId` mismatch;
- approval marker mismatch;
- session is not `2af4bfb6-a20e-453e-9804-1b8c0afbdd68`;
- an idempotency batch already exists as running/completed.
